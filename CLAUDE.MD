## Database: SQLite

All app data is stored locally in SQLite. No external database. No sync service.

Schema will be defined when development begins. Anticipated tables: `notes`, `attachments`, `audio_files`, `trash`, `chat_history`, `settings`.

## App Structure — 4 Pages

---

### 1. Home Page

**Purpose:** Main landing screen. Displays all notes as cards.

#### Components

**Search Bar**

- Positioned at the top of the page
- Searches across note titles and content

**Note Cards (List)**

- Each card contains:
  - Notes icon
  - Title — auto-generated by AI
  - Snippet — preview of note content
  - Timestamp — creation time or last edited time
  - Ellipsis icon (⋯) — "More" menu per card
    - Options: **Move to Trash**, **Rename**

**Card List Grouping**

- Cards are grouped by creation date OR last edited date (same logic)
- Groups:
  - Today
  - Yesterday
  - Previous Week
  - Previous Month
  - _(and so on for older dates)_

**Bottom Menu Bar**

- Three items:
  1. **Home** — navigates to Home Page
  2. **+** — opens New Note Page
  3. **ChatAI** — navigates to AI Chat Page

**Top-Right More Button**

- Ellipsis icon (⋯) at the top-right of the page
- Navigates to **Settings/More Page**

---

### 2. New Note Page / Note Detail Page

> **Note:** New Note and Note Detail are the **same page**. Opening an existing note loads its content into this page. Creating a new note opens this page blank and immediately shows the keyboard.

**Purpose:** Create or edit a note.

#### Core Behavior

- Keyboard appears immediately on open
- Auto-generated title (AI-generated from content)
- "Last updated" timestamp displayed below the title

#### Top-Right More Menu

- Share (copy link)
- Move to Trash
- Rename

#### Toolbar (above keyboard)

**Input / Attach**
| Icon | Action |
|---|---|
| 🎙 Audio | Record audio → can transcribe, rename audio file, download audio file, delete audio file |
| 📷 Camera | Opens camera directly to capture photo |
| 🖼 Image | Opens device gallery |
| 📎 File | Opens file picker; attached files can be deleted |

**Formatting — `Aa`**

- Headings: H1, H2, H3
- Bold, Italic, Underline, Strikethrough
- Indent left, Indent right
- Inline code
- Code block
- Block quote
- Divider (horizontal rule)

**List**

- Bullet list
- Checklist (checkbox)
- Numbered list (1. 2. 3.)

**Table**

- Insert table

**Edit History**

- Undo
- Redo

#### Slash & Shortcut Commands

- Typing `/` or pressing `+` → opens full formatting/command menu (same list as toolbar)
- Typing `@` → opens note-linking picker; shows list of existing notes to link to

---

### 3. AI Chat Page

**Purpose:** On-device conversational AI interface powered by QVAC SDK.

#### Components

**Search / Input Bar**

- `+` button inside bar → attach options:
  - Image (from gallery)
  - Camera (capture new)
  - File

**Mic Button**

- Records audio question for voice input

**Top-Right Icons**

- **History** — opens list of past chat sessions
- **New Chat** (square-and-pencil icon) — starts a fresh chat session

**Suggested Questions (Cards)**

- A set of recommendation cards displayed when the chat is empty
- Shows sample questions the user can tap to ask

**Anticipated sections (not final):**

## Key Design Constraints

- **Figma is the source of truth for all visual decisions.** Do not guess spacing, colors, typography, or layout. Wait for the Figma link.
- Touch targets must meet platform accessibility minimums (44pt on iOS).
- All screens must support both light and dark mode (exact implementation per Figma).
- The app must be fully functional offline — no feature should require a network connection.

All AI runs entirely on-device. Planned uses:

- **Audio transcription** — converts recorded audio to text within a note
- **AI Chat** — general-purpose on-device assistant (Chat Page)
